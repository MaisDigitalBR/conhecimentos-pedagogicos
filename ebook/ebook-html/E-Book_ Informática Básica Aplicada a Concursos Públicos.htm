🧩 CAPÍTULO 1: NOÇÕES DE INFORMÁTICA APLICADAS AO CONTEXTO EDUCACIONAL

1.1 Conceito de Informática

A Definição Clássica

O termo Informática nasceu da junção de duas palavras: Informação + Automática.

●     Conceito: É a ciência que estuda o tratamento racional e automático da informação por meio de computadores.

Para a IBADE, é vital entender que a informática não é apenas o "computador físico", mas o processo de transformar dados brutos em conhecimento útil de forma veloz e organizada.

Informática no Contexto Escolar

No ambiente da SEDUC/RO, a informática deixa de ser apenas uma ferramenta de escritório e passa a ser um instrumento de mediação pedagógica. Ela é dividida em duas grandes frentes que você precisa diferenciar para a prova:

  1. Informática Administrativa: Refere-se ao uso da tecnologia para a gestão da escola. Exemplos: controle de estoque da merenda, gestão de matrículas, folha de pagamento dos professores e emissão de históricos escolares.
  2. Informática Educacional (ou Pedagógica): É o uso do computador como ferramenta de apoio ao processo de ensino-aprendizagem. O foco aqui não é "aprender sobre o computador", mas "aprender COM o computador" (Softwares educativos, simuladores de laboratório, pesquisas guiadas).

1.2 Componentes Básicos de um Sistema Computacional

Um sistema computacional só funciona plenamente se houver a integração harmônica de três pilares: Hardware, Software e Peopleware.

A. Hardware (A parte física)

É todo componente tangível, que você pode tocar.

●     Exemplos: Processador (CPU), Memória RAM, Disco Rígido (HD/SSD), Teclado, Monitor, Placa-mãe.

●     Dica para a prova: A IBADE adora classificar periféricos (Entrada, Saída e Entrada/Saída). Veremos isso detalhadamente nos próximos capítulos, mas lembre-se: Hardware é o corpo.

B. Software (A parte lógica)

É o conjunto de instruções que dizem ao hardware o que fazer. São os programas, aplicativos e o sistema operacional.

●     Software de Sistema: O principal é o Sistema Operacional (ex: Windows, Linux). Sem ele, o hardware é inútil.

●     Software Aplicativo: Programas com funções específicas para o usuário (ex: Word, Excel, Navegadores).

●     Software Utilitário: Programas de manutenção e proteção (ex: Antivírus, Compactadores de arquivos como WinRAR).

C. Peopleware (O elemento humano)

Muitas vezes esquecido pelos candidatos, o Peopleware representa as pessoas que utilizam, programam ou mantêm os sistemas.

●     No seu concurso, você (o Professor) e os alunos são os principais componentes do Peopleware no ecossistema da SEDUC.

D. Dados vs. Informação (O "Pulo do Gato")

A banca pode tentar confundir esses dois conceitos:

●     Dado: É o elemento bruto, sem tratamento. Exemplo: O número "35". (Sozinho, ele não diz nada).

●     Informação: É o dado processado e com significado. Exemplo: "A temperatura na sala de aula é 35°C".

Resumo: A Informática pega os Dados, processa-os e entrega Informação.

1.3 Informática no Cotidiano do Professor

A modernização da SEDUC/RO exige que o docente domine ferramentas digitais para otimizar seu tempo e melhorar a didática.

●     Planejamento de Aulas: Uso de editores de texto (Word/Google Docs) para planos de ensino e softwares de apresentação (PowerPoint/Canva) para tornar as aulas mais atrativas.

●     Registro de Notas e Frequência: Migração do diário de classe de papel para sistemas online (Diário de Classe Digital). Aqui, o domínio de planilhas eletrônicas (Excel) é um diferencial citado em editais.

●     Comunicação Institucional: Uso ético e funcional de e-mails corporativos e plataformas de mensagens para alinhamento com a coordenação pedagógica.

●     Plataformas Educacionais e AVAs: O uso de Ambientes Virtuais de Aprendizagem (como o Moodle ou Google Classroom) tornou-se essencial. O professor precisa saber postar materiais, criar fóruns e gerenciar atividades digitais.

🖥️ CAPÍTULO 2: SISTEMAS OPERACIONAIS EM AMBIENTE WINDOWS

O Sistema Operacional (SO) é o software mais importante do computador. Sem ele, a máquina é apenas um amontoado de metal e plástico. Para a IBADE, o SO é o "gerente" que controla o hardware e permite que o usuário interaja com o sistema.

2.1 Conceito de Sistema Operacional

O SO atua como uma interface entre o Hardware e o Usuário. Suas principais funções são:

●     Gerenciamento de Processos: Coordena quais programas estão rodando e quanta atenção da CPU eles recebem.

●     Gerenciamento de Memória: Aloca espaço na memória RAM para os aplicativos.

●     Gerenciamento de Arquivos: Organiza como os dados são salvos no HD ou SSD.

Exemplos e Interfaces

  1. Windows: O mais comum em órgãos públicos (SEDUC/RO). É um software proprietário (pago).
  2. Linux: Um sistema de código aberto (open source). Veremos suas particularidades no fim deste capítulo.
  3. macOS: Sistema exclusivo da Apple.
  4. Interface Gráfica (GUI): Usa ícones, janelas e mouse (foco da prova).
  5. Linha de Comando (CLI): O usuário digita comandos de texto (ex: Prompt de Comando ou Terminal).

2.2 Ambiente Windows (Foco em Windows 10 e 11)

Embora o Windows 11 seja o mais atual, a IBADE costuma focar em recursos que são comuns a ambos.

●     Área de Trabalho (Desktop): A tela principal. É tecnicamente uma pasta dentro do sistema.

●     Barra de Tarefas: Localizada por padrão na parte inferior. Contém o Botão Iniciar, ícones fixados e a Área de Notificação (relógio, rede, volume).

●     Menu Iniciar: O "coração" do Windows. No Windows 11, ele é centralizado por padrão; no Windows 10, fica à esquerda.

2.3 Interface Gráfica e Janelas

No Windows, tudo ocorre dentro de "janelas". Você deve dominar os controles básicos:

●     Minimizar: Esconde a janela na barra de tarefas.

●     Maximizar: Faz a janela ocupar a tela inteira.

●     Restaurar: Retorna a janela ao seu tamanho anterior (substitui o botão de maximizar quando a tela está cheia).

●     Fechar: Encerra o programa.

⌨️ Atalho Essencial

●     Alt + Tab: Alterna rapidamente entre as janelas abertas.

●     Windows + D: Minimiza todas as janelas e mostra a Área de Trabalho imediatamente.

2.4 Gerenciamento de Janelas e Multitarefas

A IBADE adora cobrar recursos de produtividade modernos:

  1. Snap Layouts (Organização Lado a Lado): Arrastar uma janela para o canto da tela para que ela ocupe exatamente metade ou um quarto da tela.
  2. Área de Trabalho Virtual: Permite criar "múltiplas telas" para organizar o trabalho (ex: uma tela para o Diário de Classe e outra para pesquisa na internet).

○     Atalho: Windows + Tab (Visão de Tarefas).

2.5 Configurações do Sistema

Aqui mora o maior perigo para o concurseiro. O Windows possui dois caminhos para configurações:

1. Painel de Controle (O Clássico)

Ainda presente para funções legadas e avançadas.

●     O que você encontra lá: Desinstalar programas, Contas de Usuário (estilo antigo), Central de Rede e Compartilhamento.

2. Aplicativo "Configurações" (O Moderno)

Interface simplificada e touch-friendly.

●     Sistema: Resolução de tela, som, notificações.

●     Dispositivos: Instalar impressoras, mouses Bluetooth.

●     Hora e Idioma: Ajustar fuso horário (importante para Rondônia devido ao fuso -1h em relação a Brasília).

2.6 Atualizações do Sistema (Windows Update)

⚠️ Atenção: A IBADE sabe que muita gente acha que atualização é apenas para deixar o sistema bonito. Errado!

●     Importância: A principal função do Windows Update é a Segurança. Ele baixa "patches" (correções) para vulnerabilidades que vírus poderiam explorar.

●     Atualizações Automáticas: O Windows 10/11 força atualizações críticas para garantir a integridade do sistema.

●     Diferença: Atualizações de Recursos (novas funções) vs. Atualizações de Qualidade/Segurança (correção de erros).

2.7 Ambiente Linux (O "Terror" dos Candidatos)

A IBADE pode cobrar noções básicas de Linux para comparar com o Windows.

●     Kernel: É o núcleo do sistema (a parte que fala com o hardware).

●     Distribuições (Distros): Como o Linux é aberto, existem várias "versões" como Ubuntu, Debian, Fedora e Mint.

●     Case Sensitive: No Linux, arquivos "Aula.txt" e "aula.txt" são diferentes. No Windows, são o mesmo.

●     Principais Comandos (CLI):

○     ls: Listar arquivos.

○     cd: Mudar de diretório.

○     mkdir: Criar pasta.

○     pwd: Mostra em qual pasta você está agora.

📌 Pegadinhas Comuns da IBADE

1. Painel de Controle vs. Configurações

A banca pode afirmar que o "Painel de Controle foi totalmente removido do Windows 11". Falso. Ele ainda existe, embora a Microsoft esteja migrando as funções gradualmente.

2. A Finalidade das Atualizações

Uma questão clássica diz que "as atualizações do Windows servem apenas para melhorar a interface gráfica". Falso. O foco principal, para fins de prova, é segurança e correção de bugs.

3. Gerenciamento de Arquivos

A banca pode perguntar se é possível restaurar uma janela através do Menu Iniciar. Falso. A restauração é feita na própria janela ou via Barra de Tarefas.

Para a prova da SEDUC, lembre-se: o Windows é um sistema Multitarefa (faz várias coisas ao mesmo tempo) e Multiusuário (várias pessoas podem ter contas diferentes no mesmo computador).

📁 CAPÍTULO 3: EXPLORADOR DE ARQUIVOS E GERENCIAMENTO DE DADOS

Para o professor da rede estadual, organizar planos de aula, provas e registros de alunos exige um domínio pleno da estrutura de arquivos do Windows. A IBADE cobra este conhecimento sob a ótica da organização lógica e das regras de nomenclatura.

3.1 Explorador de Arquivos do Windows

Antigamente chamado de Windows Explorer, o Explorador de Arquivos é o gerenciador oficial de pastas e arquivos do sistema.

Acesso ao Explorador

Existem três formas clássicas de acessá-lo, mas apenas uma cai massivamente em prova:

●     ⌨️ Atalho de Ouro: Windows + E (E de Explorer).

●     Barra de Tarefas: Clicando no ícone da "pasta amarela".

●     Menu Iniciar: Pesquisando por "Explorador de Arquivos".

Estrutura de Diretórios e Unidades

O Windows organiza os dados em uma hierarquia de "árvore".

  1. Unidades (Raiz): Identificadas por letras seguidas de dois pontos.

○     C: -> Geralmente o Disco Local (onde o Windows está instalado).

○     D:, E: -> Podem ser partições de backup, Pen Drives ou HDs externos.

  1. Pastas e Subpastas: Recipientes lógicos para organizar arquivos.
  2. Arquivos: A unidade básica de informação (o documento em si).

3.2 Pastas e Arquivos: Regras e Operações

Criação e Renomeação

Para criar uma pasta, podemos usar o botão direito ou o atalho Ctrl + Shift + N.

⚠️ ALERTA DE PEGADINHA (Caracteres Proibidos):

A IBADE ama perguntar quais caracteres NÃO podem ser usados em nomes de arquivos ou pastas. O Windows proíbe 9 caracteres. Decore o "bonequinho" ou a lista:

●     \ (Barra invertida)

●     / (Barra inclinada)

●     : (Dois pontos)

●     * (Asterisco)

●     ? (Ponto de interrogação)

●     " (Aspas duplas)

●     < (Menor que)

●     > (Maior que)

●     | (Pipe ou barra vertical)

Cópia vs. Movimentação

●     Copiar (Ctrl + C / Ctrl + V): Cria uma duplicata. O original permanece.

●     Recortar/Mover (Ctrl + X / Ctrl + V): Tira de um lugar e coloca em outro. O original deixa de existir no local de origem.

Dica IBADE: Se você arrastar um arquivo com o mouse para uma pasta na mesma unidade (ex: de C:\Documentos para C:\Provas), ele MOVE. Se arrastar para uma unidade diferente (ex: do C: para um Pen Drive D:), ele COPIA.

3.4 Organização e Pesquisa

Ordenação

No Explorador, você pode ordenar arquivos por:

●     Nome: Ordem alfabética.

●     Data de Modificação: Úteis para encontrar o arquivo que você acabou de salvar.

●     Tipo: Agrupa todos os PDFs juntos, todos os Word juntos, etc.

●     Tamanho: Identifica arquivos que estão ocupando muito espaço.

Pesquisa de Arquivos

O Windows permite usar "Curingas":

●     *: Substitui vários caracteres. (Ex: *.docx busca todos os arquivos Word).

●     ?: Substitui apenas um caractere.

3.5 Administração Básica de Usuários

No ambiente escolar (SEDUC), é comum que um computador seja usado por vários professores. Por isso, existem tipos de contas:

  1. Usuário Administrador: Tem controle total. Pode instalar programas, excluir arquivos de sistema e criar outros usuários.
  2. Usuário Padrão: Pode usar o computador e salvar seus arquivos, mas não pode instalar softwares que afetem outros usuários ou alterar configurações críticas de segurança.

Importância Pedagógica: Em laboratórios de informática, os alunos devem usar contas "Padrão" para evitar que desconfigurem as máquinas ou instalem softwares indevidos.

📌 Cobrança Típica da IBADE

1. Extensão ≠ Formato ≠ Programa

A banca pode dizer que ".docx é um programa". Errado! .docx é a extensão do formato de arquivo gerado pelo programa Microsoft Word.

2. Permissões de Administrador

Questões costumam perguntar: "O professor deseja instalar um novo antivírus, mas o sistema solicita uma senha".

●     Explicação: O professor está logado como Usuário Padrão e a instalação exige privilégios de Administrador.

3. A Lixeira

A IBADE adora perguntar: "Ao excluir um arquivo de um Pen Drive, ele vai para a Lixeira?".

●     Resposta: NÃO. Arquivos excluídos de unidades removíveis (Pen Drives, HDs externos) são excluídos permanentemente. A Lixeira só armazena arquivos excluídos do HD/SSD interno.

●     Atalho para exclusão permanente: Shift + Delete.

💡 DICA DE MESTRE

Grave os 9 caracteres proibidos. Uma técnica comum é lembrar da frase: "Baseado no que você viu, o bonequinho está de castigo no 'pipi' ( | )". Ou simplesmente visualize que você não pode usar símbolos que o sistema usa para caminhos de diretórios (como / e \) ou para perguntas (?).

Para o professor da SEDUC/RO, as suítes de escritório são as ferramentas de trabalho mais utilizadas após o navegador de internet. A banca IBADE costuma cobrar a comparação entre essas ferramentas, focando em compatibilidade, extensões de arquivos e a diferença de atalhos entre elas.

4.1 Conceito de Suítes de Escritório

Uma Suíte de Escritório (ou Office Suite) é um conjunto de aplicativos integrados que compartilham uma interface semelhante e permitem a troca de dados entre si de forma facilitada.

●     Microsoft Office (agora Microsoft 365): A suíte comercial mais famosa. Proprietária (paga) e padrão no mercado Windows.

●     LibreOffice: Uma suíte gratuita e de código aberto. É a sucessora do antigo BrOffice. Muito cobrada em concursos por ser a alternativa oficial ao Windows em órgãos que utilizam Linux.

●     Google Workspace (antigo G Suite): Suíte baseada em computação em nuvem. Não precisa de instalação local, rodando diretamente no navegador.

4.2 Aplicações Educacionais

No cotidiano docente, cada ferramenta da suíte possui uma finalidade específica:

  1. Processadores de Texto: Criação de provas, planos de aula, apostilas e projetos pedagógicos.
  2. Planilhas Eletrônicas: Cálculo de médias, controle de frequência, gráficos de desempenho escolar e cronogramas.
  3. Softwares de Apresentação: Criação de slides para aulas expositivas, reuniões de pais e conselhos de classe.

Principais Ferramentas:

📊 CAPÍTULO 6: MICROSOFT EXCEL (PLANILHAS ELETRÔNICAS)

Se o Word é o padrão para documentos, o Excel é a ferramenta universal para controle de dados. No contexto da SEDUC/RO, ele é vital para calcular médias de turmas, estatísticas de evasão e cronogramas de carga horária. Para a prova, o Excel não é um editor de tabelas, mas uma calculadora lógica.

6.1 Conceitos Básicos: A Estrutura do Excel

Para não cair em pegadinhas de nomenclatura, memorize a hierarquia:

  1. Pasta de Trabalho: É o arquivo em si (extensão padrão .xlsx). Um arquivo do Excel pode conter várias planilhas.
  2. Planilha: É a "folha" de cálculo (as abas que ficam na parte inferior).
  3. Célula: É a unidade básica, o encontro de uma linha com uma coluna.
  4. Colunas: São identificadas por Letras (A, B, C... Z, AA...). Elas ficam na vertical.
  5. Linhas: São identificadas por Números (1, 2, 3...). Elas ficam na horizontal.
  6. Endereçamento: A "identidade" de uma célula é sempre Letra + Número (Ex: B5 é a célula na coluna B, linha 5).

6.2 Formatação de Planilhas e Tipos de Dados

O Excel precisa saber o que tem dentro da célula para calcular corretamente.

●     Tipos de Dados (Guia Página Inicial):

○     Geral: Sem formato específico.

○     Número: Para cálculos matemáticos.

○     Moeda / Contábil: Para valores em dinheiro (R$).

○     Data/Hora: Essencial para cronogramas.

○     Porcentagem: Multiplica o valor por 100 e exibe o símbolo %.

●     Ajuste de Colunas: Se uma célula mostrar vários símbolos de sustenido (#######), não é um erro de cálculo; apenas significa que a coluna está estreita demais para exibir o valor. Basta dar um clique duplo na divisória da coluna para autoajustar.

6.3 Fórmulas e Funções Básicas (O Pulo do Gato)

⚠️ REGRA DE OURO: Toda fórmula ou função no Excel DEVE começar com o sinal de igual (=). Se começar com qualquer outra coisa, o Excel tratará como texto simples.

A. Operadores Matemáticos

●     Adição (+) | Subtração (-) | Multiplicação (*) | Divisão (/) | Exponenciação (^).

B. O Segredo dos Dois Pontos (:) e do Ponto e Vírgula (;)

A IBADE ama confundir isso:

●     Dois pontos (:): Significa ATÉ. Seleciona um intervalo contínuo.

○     Ex: =SOMA(A1:A3) -> Soma A1, A2 e A3.

●     Ponto e Vírgula (;): Significa E. Seleciona células isoladas.

○     Ex: =SOMA(A1;A3) -> Soma apenas A1 e A3.

C. Funções mais cobradas pela IBADE:

  1. =SOMA(intervalo): Soma todos os valores.
  2. =MÉDIA(intervalo): Soma os valores e divide pela quantidade de células.
  3. =MÍNIMO(intervalo): Retorna o menor valor da lista.
  4. =MÁXIMO(intervalo): Retorna o maior valor da lista.
  5. =CONT.SE(intervalo; critério): Conta quantas células atendem a uma condição.

○     Ex: =CONT.SE(B1:B10; ">6") -> Conta quantos alunos tiraram nota maior que 6.

6.4 Gráficos: Visualização de Dados

Transformar números em imagens é essencial para relatórios pedagógicos.

●     Gráfico de Colunas/Barras: Ideal para comparar valores entre categorias (ex: notas de diferentes turmas).

●     Gráfico de Pizza: Ideal para mostrar partes de um todo (ex: porcentagem de alunos aprovados vs. reprovados). Atenção: Não use Pizza para muitos dados.

●     Gráfico de Linhas: Ideal para mostrar tendências ao longo do tempo (ex: evolução da nota de um aluno durante o ano).

📌 IBADE Cobra Muito: Pegadinhas e Detalhes Técnicos

1. Diferença entre Fórmula e Função

●     Fórmula: É criada manualmente pelo usuário. Ex: =A1+A2+A3.

●     Função: É um comando pré-definido do Excel. Ex: =SOMA(A1:A3).

●     Ambas dão o mesmo resultado, mas a função é mais eficiente para grandes volumes de dados.

2. Endereçamento Relativo vs. Absoluto (O terror das provas)

Quando você copia uma fórmula para outra célula, o Excel ajusta as referências automaticamente (Relativo). Para "travar" uma célula e impedir que ela mude na cópia, usamos o cifrão ($) (Absoluto).

●     A1 -> Relativo (muda se arrastar).

●     $A$1 -> Absoluto (fica travado na Coluna A e na Linha 1).

●     A$1 -> Misto (trava apenas a linha).

3. A Função MÉDIA com células vazias

Se uma célula estiver vazia, a função =MÉDIA a ignora. Se a célula tiver o número 0, ela entra no cálculo. Isso altera o resultado final!

🌐 CAPÍTULO 8: REDES DE COMPUTADORES

Uma rede de computadores não é apenas um conjunto de cabos. É um sistema complexo de hardware e regras (protocolos) que permite a troca de informações. Para o concurso da SEDUC/RO, você deve focar na classificação geográfica e na função dos equipamentos.

8.1 Conceitos Básicos

O que é uma Rede?

É um conjunto de dois ou mais dispositivos (computadores, celulares, impressoras, servidores) interconectados para compartilhar recursos e informações.

Finalidade

●     Compartilhamento de Hardware: Vários computadores usando a mesma impressora da escola.

●     Compartilhamento de Dados: Professores acessando arquivos em um servidor central.

●     Comunicação: E-mail, videoaulas e sistemas de gestão escolar.

●     Acesso à Internet: Distribuição do sinal de internet para todos os setores.

8.2 Tipos de Redes (Classificação Geográfica)

A IBADE ama classificar as redes pelo seu alcance. Memorize esta escala:

8.3 Dispositivos de Rede: Quem faz o quê?

Muitos candidatos confundem esses três aparelhos. Entenda a função de cada um:

  1. Modem: É o "tradutor". Ele recebe o sinal da operadora (fibra, cabo ou linha telefônica) e o converte em um sinal que o computador entende. Sem ele, não há conexão externa.
  2. Roteador: É o "guarda de trânsito". Ele escolhe a melhor rota para os dados e distribui a internet para vários dispositivos (via cabo ou Wi-Fi). Ele interliga a sua rede local (LAN) com a internet (WAN).
  3. Switch (Comutador): É o "organizador interno". Ele conecta vários aparelhos dentro da mesma rede local. O Switch é inteligente: ele envia os dados apenas para o computador de destino, evitando sobrecarga.

⚠️ Alerta de Pegadinha: A banca pode citar o HUB. O Hub é um ancestral "burro" do Switch. Ele envia a informação para todos os computadores da rede ao mesmo tempo, causando lentidão. Se a questão falar em "conectar dispositivos em estrela com inteligência", a resposta é Switch.

8.4 Protocolos: A Linguagem da Rede

Computadores de marcas diferentes só se entendem porque falam a mesma "língua": os Protocolos.

●     TCP/IP: É o conjunto de protocolos base da Internet. O IP (Internet Protocol) é o endereço único de cada máquina na rede.

●     HTTP vs. HTTPS:

○     HTTP: Protocolo de transferência de hipertexto (sites). Não é seguro.

○     HTTPS: É o HTTP com uma camada de segurança (SSL/TLS). Os dados são criptografados. Essencial para bancos e sistemas da SEDUC.

🌍 CAPÍTULO 9: INTERNET, INTRANET E EXTRANET

Muitos candidatos acreditam que a diferença entre essas redes é a tecnologia utilizada (cabos, satélite ou Wi-Fi). Erro fatal! A diferença reside no acesso, no público-alvo e na propriedade da rede. As tecnologias e protocolos (TCP/IP, HTTP, etc.) são exatamente os mesmos.

9.1 Internet: A Rede das Redes

A Internet é um conglomerado global de redes interconectadas. Ela é pública, descentralizada e ninguém "é dono" dela como um todo.

●     Conceito: É uma rede mundial (WAN - Wide Area Network) que utiliza a suíte de protocolos TCP/IP para permitir a comunicação entre dispositivos em qualquer lugar do planeta.

●     Serviços da Internet:

○     WWW (World Wide Web): É apenas um dos serviços (o acesso a sites). Não confunda Internet com Web!

○     E-mail: Transferência de mensagens eletrônicas.

○     FTP: Protocolo para transferência de arquivos.

○     Streaming: Consumo de áudio e vídeo em tempo real (YouTube, Spotify).

○     Computação em Nuvem: Armazenamento e processamento de dados em servidores remotos.

9.2 Intranet: O Coração da Instituição

A Intranet é uma rede privada que pertence a uma organização (como a SEDUC/RO ou uma escola específica).

●     Conceito: É uma rede que utiliza as mesmas tecnologias da Internet (navegadores, protocolos, servidores web), mas o seu acesso é restrito a um público interno (funcionários, professores e gestores).

●     Uso Institucional e Escolar:

○     Comunicação Interna: Mural de avisos, circulares da diretoria e memorandos.

○     Sistemas de Gestão: Acesso ao diário de classe eletrônico, lançamento de notas e portal do servidor.

○     Compartilhamento de Recursos: Repositório de planos de aula e materiais didáticos exclusivos daquela escola.

●     Segurança: A Intranet geralmente é protegida por um Firewall, que impede que pessoas de fora da rede consigam acessar os dados internos.

9.3 Extranet: A Ponte para o Mundo Externo

A Extranet é, de forma simplificada, uma extensão da Intranet para usuários autorizados que estão fora da rede física da instituição.

●     Conceito: É uma rede que permite o acesso externo controlado a uma parte da Intranet.

●     Acesso Controlado: Para acessar a Extranet, o usuário precisa de credenciais (Login e Senha) e, muitas vezes, de uma conexão segura via VPN (Virtual Private Network).

●     Exemplos Práticos na SEDUC:

○     O professor que, de sua casa (Internet), acessa o sistema de frequência da escola (Intranet) está utilizando uma Extranet.

○     Pais de alunos que acessam o portal para ver boletins estão acessando uma área de Extranet da instituição.

📧 CAPÍTULO 11: CORREIO ELETRÔNICO

O correio eletrônico (e-mail) é um dos serviços mais antigos e essenciais da internet. No concurso da SEDUC/RO, a banca focará na sua capacidade de distinguir as formas de acesso, os protocolos envolvidos e, principalmente, as regras de envio para múltiplos destinatários.

11.1 Conceitos Básicos

O e-mail é um sistema de troca de mensagens assíncronas (quem recebe não precisa estar online no momento do envio).

Cliente de E-mail vs. Webmail

Este é o ponto que mais gera confusão em provas. Existem duas formas de gerenciar seus e-mails:

  1. Webmail: É quando você acessa seus e-mails através de um navegador (Chrome, Edge). Exemplos: Gmail, Outlook.com, Yahoo Mail.

○     Vantagem: Acessível de qualquer computador com internet. As mensagens ficam no servidor.

  1. Cliente de E-mail (Software): É um programa instalado no computador ou celular especificamente para isso. Exemplos: Microsoft Outlook (do pacote Office), Mozilla Thunderbird.

○     Vantagem: Recursos avançados de organização offline, calendários complexos e gestão de múltiplas contas em uma única interface.

11.2 Microsoft Outlook e Gmail (Funcionalidades)

Embora um seja software e o outro seja majoritariamente usado como Webmail, as funcionalidades cobradas pela IBADE são semelhantes:

●     Envio e Recebimento: O processo básico. No Outlook, existe o botão "Enviar/Receber" para forçar a sincronização.

●     Anexos: Arquivos enviados junto com a mensagem.

○     ⚠️ Cuidado: Existe um limite de tamanho (geralmente entre 20MB e 25MB). Para arquivos maiores, os sistemas sugerem o uso de links da nuvem (Google Drive ou OneDrive).

○     Ícone: Representado universalmente por um Clipe de papel.

●     Pastas e Marcadores: No Outlook usamos "Pastas" (hierárquicas). No Gmail, usamos "Marcadores" (uma mensagem pode ter vários marcadores ao mesmo tempo).

●     Contatos e Calendário: Ambos integram uma agenda de contatos e um calendário/agenda para marcação de reuniões e conselhos de classe.

●     Spam (Lixo Eletrônico): Mensagens não solicitadas, geralmente enviadas em massa. Os filtros de IA tentam desviar essas mensagens da Caixa de Entrada automaticamente.

11.3 Estrutura de Envio (Onde a IBADE mais pontua)

Ao redigir um e-mail, você tem três campos de destinatários. A banca AMA cobrar isso:

  1. Para (To): Destinatário principal.
  2. Cc (Cópia Carbono): Destinatários que devem ter conhecimento do e-mail, mas não são os principais. Todos os que recebem veem os endereços uns dos outros.
  3. Cco (Cópia Carbono Oculta / BCC): Os endereços listados aqui não são vistos pelos outros destinatários.

○     Dica Pedagógica: Ideal para enviar um comunicado a todos os pais de alunos sem expor o e-mail privado de uma família para a outra.

11.4 Boas Práticas e Segurança

Linguagem e Netiqueta

No serviço público (SEDUC), o e-mail é um documento oficial.

●     Assunto: Deve ser claro e resumir o conteúdo (ex: "Relatório de Notas - 3º Ano B"). Nunca deixe o assunto em branco.

●     Linguagem: Formal, evitando gírias, "internetês" e o uso de CAIXA ALTA (que na internet equivale a gritar).

Segurança e Protocolos (Recapitulação Técnica)

Como vimos no Capítulo 8, a banca pode misturar conceitos. Lembre-se:

●     SMTP: Envia.

●     POP3: Recebe e baixa (apaga do servidor).

●     IMAP: Recebe e sincroniza (mantém no servidor).

●     Phishing: Cuidado com e-mails que fingem ser de órgãos oficiais (como a Secretaria de Educação ou Bancos) pedindo para clicar em links ou baixar arquivos .exe.

📌 Como a IBADE cobra este capítulo?

1. A Pegadinha do "Responder a Todos"

"João recebeu um e-mail enviado para ele, com Maria em Cc e José em Cco. Se João clicar em 'Responder a Todos', quem receberá a resposta?"

●     Resposta: Apenas o Remetente original e a Maria (Cc). O José (Cco) nunca recebe respostas automáticas de "Responder a Todos" e ninguém sabe que ele recebeu o e-mail original.

2. Diferença entre Cliente e Webmail

A banca pode dizer que para usar o Gmail é obrigatório ter o Microsoft Outlook instalado. Falso. O Gmail é um Webmail e funciona direto no navegador.

3. Gerenciamento de Anexos

Questões podem perguntar se é possível enviar uma pasta inteira por e-mail sem compactá-la.

●     Resposta: Geralmente não. Você deve compactar a pasta (formato .zip ou .rar) ou enviar os arquivos um a um.

💡 DICA DE MESTRE

Grave as siglas: Cco = Oculto. Se caiu na prova que alguém quer enviar um e-mail para 50 pessoas sem que elas saibam quem mais recebeu, a solução é colocar os 50 endereços no campo Cco.

12.4 Ética e Uso Responsável da Informação

Aqui a informática se une ao Direito e à Pedagogia.

A. Plágio

O plágio ocorre quando alguém apresenta uma obra, trecho ou ideia de outrem como se fosse sua, sem dar o devido crédito.

●     Na Escola: O professor deve orientar os alunos a citar fontes.

●     Na Prova: A IBADE pode perguntar sobre ferramentas de "detecção de plágio" (softwares que comparam textos com a base da internet).

B. Direitos Autorais e Propriedade Intelectual

Toda obra original (texto, foto, música) é protegida por lei.

●     Copyright (©): "Todos os direitos reservados". Você precisa de autorização expressa do autor para usar.

●     Creative Commons (CC): Licenças mais flexíveis. O autor permite o uso, desde que seguidas certas regras (como citar o nome ou não usar para fins comerciais).

●     Domínio Público: Obras cujo prazo de proteção expirou (geralmente 70 anos após a morte do autor no Brasil) ou que foram doadas à coletividade.

As redes sociais deixaram de ser apenas lazer para se tornarem canais de comunicação institucional e ferramentas pedagógicas. Para a prova, você deve entender a estrutura dessas redes e como elas se comportam em um ambiente controlado como a escola.

13.1 Conceitos e Funcionalidades

Redes Sociais vs. Mídias Sociais

Embora usados como sinônimos, a teoria da comunicação faz uma distinção que pode aparecer na sua prova:

●     Redes Sociais: Focam nas conexões e relacionamentos entre pessoas ou organizações (o "nó" da rede). Ex: Facebook, LinkedIn.

●     Mídias Sociais: Focam na estratégia de difusão de conteúdo (um para muitos). Ex: YouTube, TikTok.

Principais Funcionalidades para o Setor Público

  1. Comunicação Síncrona: Acontece em tempo real (Ex: Lives no Instagram, chats instantâneos no WhatsApp).
  2. Comunicação Assíncrona: Não exige resposta imediata (Ex: Postagens no feed, comentários em fóruns).
  3. Algoritmos de Relevância: São as regras lógicas que determinam o que aparece primeiro para o usuário. A IBADE pode cobrar o conceito de "bolhas digitais" (quando o algoritmo só mostra o que o usuário já gosta).

13.2 Impactos Educacionais: O Papel do Professor

O uso de redes sociais na educação é uma faca de dois gumes. A banca costuma cobrar a capacidade do candidato de identificar vantagens e riscos.

Vantagens Pedagógicas

●     Engajamento: Utiliza uma linguagem familiar ao aluno (nativos digitais).

●     Colaboração: Facilita a criação de grupos de estudo e troca de materiais extraclasse.

●     Extensão da Sala de Aula: Permite que o debate continue além do horário escolar.

●     Desenvolvimento da Autoria: Estimula o aluno a produzir conteúdo (vídeos, textos, artes) em vez de apenas consumir.

Riscos e Desafios

●     Ciberbullying: Prática de intimidação, humilhação ou exposição vexatória no ambiente digital. A escola tem responsabilidade legal de atuar nesses casos.

●     Distração: O uso sem objetivos claros pode dispersar o foco do conteúdo programático.

●     Fake News (Desinformação): A rapidez das redes propaga notícias falsas. O professor deve atuar como um mediador de letramento digital.

13.3 Privacidade e Segurança (O Foco da IBADE)

Como futuro servidor da SEDUC/RO, você deve estar atento à Lei Geral de Proteção de Dados (LGPD - Lei 13.709/2018).

Dados Pessoais e Sensíveis

●     Privacidade: É o direito do usuário de controlar quem tem acesso às suas informações.

●     Pegada Digital (Digital Footprint): É o rastro de dados que deixamos ao navegar (curtidas, fotos, geolocalização). Uma vez na rede, é quase impossível apagar totalmente.

●     Uso de Imagem de Menores: Ponto Crítico. O professor não deve postar fotos de alunos em suas redes sociais pessoais sem autorização expressa dos responsáveis e da instituição. Isso fere o ECA (Estatuto da Criança e do Adolescente) e a LGPD.

Segurança no Ambiente Escolar

●     Configurações de Privacidade: O uso de perfis "fechados" ou grupos privados para fins escolares é a prática recomendada.

●     Phishing em Redes Sociais: Links maliciosos enviados por mensagens diretas que visam roubar credenciais de acesso aos sistemas da SEDUC.

●     Verificação em Duas Etapas (2FA): Recurso de segurança indispensável para proteger contas institucionais.

1. Terminologia e Definições

☁️ CAPÍTULO 14: COMPUTAÇÃO EM NUVEM

A computação em nuvem mudou o paradigma da informática: saímos da era do "ter" (instalar softwares e salvar arquivos no HD) para a era do "acessar" (usar recursos via internet). Para a sua prova, é essencial dominar as nomenclaturas técnicas e os modelos de serviço.

14.1 Conceitos Básicos: O que é a Nuvem?

Diferente do que o nome sugere, a "nuvem" não é algo etéreo. Ela é um conjunto de servidores físicos gigantescos, localizados em data centers ao redor do mundo, que oferecem processamento e armazenamento sob demanda.

Características Essenciais (Padrão NIST)

A IBADE pode cobrar as cinco características que definem a computação em nuvem:

  1. Autoatendimento sob Demanda: O usuário pode acessar recursos (como mais espaço no Drive) sem precisar falar com um atendente humano.
  2. Acesso Amplo via Rede: O serviço está disponível via internet através de qualquer dispositivo (celular, tablet, PC).
  3. Agrupamento de Recursos: Os recursos físicos são compartilhados entre vários usuários, sendo alocados dinamicamente.
  4. Elasticidade Rápida: A capacidade de armazenamento ou processamento pode crescer ou diminuir rapidamente conforme a necessidade.
  5. Serviço Mensurado: O uso é monitorado e controlado, permitindo pagar apenas pelo que se usa (modelo de assinatura).

Modelos de Serviço (A "Sopa de Letrinhas" que cai!)

🔐 CAPÍTULO 16: SEGURANÇA DA INFORMAÇÃO E LGPD

A Segurança da Informação não se resume a ter um antivírus. É um conjunto de práticas e leis que visam proteger o maior ativo de uma instituição: a Informação. Neste capítulo, estudaremos os pilares que sustentam a proteção de dados e como a Lei Geral de Proteção de Dados (LGPD) impacta a sua sala de aula.

16.1 Princípios Básicos da Segurança (A Tríade CIA)

Para a IBADE, a segurança da informação repousa sobre três pilares fundamentais, conhecidos pela sigla CID (ou CIA, em inglês). Você precisa decorar esses conceitos, pois a banca troca as definições entre eles.

  1. Confidencialidade: Garante que a informação seja acessada apenas por pessoas autorizadas.

○     Exemplo: O diário de classe eletrônico só pode ser visualizado pelo professor da turma e pela coordenação.

  1. Integridade: Garante que a informação não foi alterada de forma indevida por pessoas ou processos não autorizados. Ela preserva a exatidão dos dados.

○     Exemplo: Se uma nota "7" for alterada para "10" sem o consentimento do professor, houve uma quebra de Integridade.

  1. Disponibilidade: Garante que a informação esteja acessível sempre que necessário pelos usuários autorizados.

○     Exemplo: Se o sistema da SEDUC/RO cai no dia de lançar as notas, houve uma quebra de Disponibilidade.

💡 Dica de Mestre: Algumas questões modernas acrescentam o pilar da Autenticidade (garantia de quem diz ser quem é) e do Não-Repúdio (impossibilidade de negar que realizou uma ação).

16.2 Senhas e Autenticação

A senha é a sua primeira linha de defesa. A IBADE costuma cobrar o que constitui uma "Senha Forte" e métodos modernos de acesso.

Boas Práticas para Senhas:

●     Comprimento: Mínimo de 8 a 12 caracteres.

●     Complexidade: Mistura de letras maiúsculas, minúsculas, números e símbolos (@, #, !).

●     Não reuso: Não usar a mesma senha para o e-mail pessoal e para o sistema da SEDUC.

●     Troca Periódica: Mudar a senha a cada 90 dias, por exemplo.

Duplo Fator de Autenticação (MFA / 2FA)

É a camada extra de segurança. Além de algo que você sabe (senha), o sistema pede algo que você tem ou é.

●     Algo que você tem: Um código enviado via SMS ou gerado por um aplicativo (Google Authenticator).

●     Algo que você é: Biometria (digital ou reconhecimento facial).

●     Por que usar? Mesmo que um hacker descubra sua senha, ele não conseguirá acessar sua conta sem o segundo fator (o código no seu celular).

A Lei Geral de Proteção de Dados regula como as instituições (incluindo a SEDUC/RO e as escolas) devem coletar, armazenar e tratar dados pessoais.

🦠 CAPÍTULO 17: AMEAÇAS DIGITAIS (MALWARES)

O termo Malware vem de Malicious Software (Software Malicioso). É qualquer programa desenvolvido para executar ações indesejadas, causar danos ou roubar informações de um sistema.

17.1 Tipos de Ameaças: A "Fauna" Cibernética

A IBADE adora cobrar as características que tornam cada malware único. Memorize estas distinções:

A. Vírus

É o tipo mais clássico, mas cuidado: a banca pode tentar dizer que tudo é vírus.

●     Característica: Ele precisa de um hospedeiro (um arquivo ou programa) para se esconder.

●     Ação: Ele só entra em execução quando o usuário abre o arquivo infectado. Ele se multiplica infectando outros arquivos do sistema.

B. Worm (Verme)

Este é o "queridinho" das provas.

●     Característica: Ele é independente (não precisa de hospedeiro).

●     Ação: Ele se autorreplica enviando cópias de si mesmo de computador para computador através das redes.

●     Foco da Prova: O Worm não foca em destruir arquivos, mas em esgotar os recursos da rede, deixando a internet da escola extremamente lenta.

C. Trojan (Cavalo de Troia)

●     Característica: Apresenta-se como um programa legítimo ou útil (um jogo, um protetor de tela, um PDF de "currículo"), mas esconde uma função maliciosa.

●     Ação: Ao ser executado, ele abre uma "porta" (backdoor) para que um invasor tome o controle do seu PC. Ele não se multiplica sozinho como o vírus.

D. Spyware (Software Espião)

Projetado para monitorar as atividades do sistema e enviar informações a terceiros.

●     Keylogger: Captura tudo o que é digitado no teclado (foco: roubo de senhas).

●     Screenlogger: Captura "fotos" (prints) da tela onde o mouse clica (foco: teclados virtuais de bancos).

●     Adware: Focado em exibir propagandas indesejadas e redirecionar pesquisas.

E. Ransomware (O sequestrador de dados)

Este é o tema mais atual para 2026.

●     Ação: Ele criptografa (tranca com senha) os arquivos do computador (planilhas de notas, fotos, documentos).

●     O Golpe: O criminoso exige um resgate (geralmente em criptomoedas) para liberar a chave de descriptografia.

●     💡 Dica de Mestre: A única solução definitiva contra o Ransomware é ter um Backup offline.

17.2 Formas de Contágio: Como o PC "fica doente"

No cotidiano da SEDUC, os vetores de infecção mais comuns são:

●     E-mail (Phishing): Mensagens falsas com anexos infectados ou links para sites fraudulentos. O criminoso usa Engenharia Social para convencer o professor a clicar.

●     Downloads de Sites Inseguros: Programas "piratas", ativadores de software ou arquivos baixados de fontes não oficiais.

●     Mídias Removíveis: Pen Drives de alunos ou colegas que circulam entre computadores infectados.

●     Vulnerabilidades do Sistema: Softwares desatualizados (sem os patches de segurança do Windows Update) que possuem "buracos" por onde os malwares entram sozinhos.

17.3 Prevenção: Sua Armadura Digital

  1. Antivírus / Antimalware: Software que detecta, bloqueia e remove ameaças. Deve estar sempre atualizado com as últimas "assinaturas" de vírus.
  2. Firewall (Parede de Fogo): É o "porteiro" da rede. Ele controla o tráfego de dados, decidindo o que pode entrar e sair do computador. Atenção: O Firewall não remove vírus; ele impede acessos não autorizados.
  3. Atualização de Software: Manter o Windows e os navegadores atualizados é a forma mais eficaz de fechar as portas para Worms.
  4. Desconfiança Ativa: Não abrir anexos de remetentes desconhecidos e verificar a URL dos sites antes de digitar dados sensíveis.

🛡️ CAPÍTULO 18: FERRAMENTAS DE SEGURANÇA

Para garantir a tríade da segurança (Confidencialidade, Integridade e Disponibilidade), o sistema operacional e o usuário utilizam ferramentas específicas. Para a prova da SEDUC/RO, o foco recai sobre o funcionamento técnico do Antivírus e do Firewall.

18.1 Antivírus (Software de Proteção)

O antivírus é um programa projetado para detectar, impedir e remover softwares maliciosos. No Windows 10 e 11, o antivírus nativo é o Microsoft Defender.

Métodos de Detecção (Cai muito!)

A IBADE pode perguntar como o antivírus sabe que um arquivo é perigoso:

  1. Assinatura: O antivírus compara o arquivo com um "banco de dados" de malwares conhecidos. É como uma lista de procurados da polícia. Limitação: Não detecta vírus criados ontem (vírus de dia zero).
  2. Heurística (Análise de Comportamento): O antivírus observa o que o programa tenta fazer. Se um arquivo tenta apagar pastas do sistema ou criptografar dados sem permissão, ele é bloqueado mesmo sem estar na lista de assinaturas.
  3. Quarentena: Quando o antivírus encontra um arquivo suspeito que não pode ser limpo com segurança, ele o move para a Quarentena — uma pasta isolada onde o vírus não pode ser executado nem infectar outros arquivos.

18.2 Firewall (A Parede de Fogo)

Diferente do antivírus, o Firewall não "caça" vírus dentro do seu computador. Ele é um mecanismo de barreira.

●     Conceito: É um software ou hardware que monitora o tráfego de dados que entra e sai de uma rede ou computador. Ele decide o que pode passar com base em regras de segurança.

●     Função: Impedir que hackers ou worms acessem seu PC através da rede e impedir que programas maliciosos instalados enviem seus dados para fora.

⚠️ ALERTA DE PEGADINHA: A IBADE vai afirmar que o Firewall serve para remover vírus de um Pen Drive. ERRADO! O Firewall não analisa o conteúdo de arquivos estáticos; ele analisa o tráfego de rede.

18.4 Atualizações Automáticas (Windows Update)

As atualizações são a ferramenta de segurança mais subestimada pelos candidatos.

●     Vulnerabilidade: É uma falha no código do programa que o fabricante ainda não corrigiu.

●     Exploit: É a ferramenta que o hacker usa para aproveitar essa falha.

●     Patch de Segurança: É o "remendo" enviado pela Microsoft via Windows Update para fechar a vulnerabilidade.

Importância: Manter o sistema atualizado garante que os "buracos" no Windows sejam fechados antes que um Worm consiga entrar.

1. Firewall vs. Antivírus

●     Questão: "Para impedir que um usuário não autorizado acesse remotamente os dados de uma escola através da internet, a ferramenta mais adequada é..."

●     Resposta: Firewall. (Se a questão falasse em "eliminar um arquivo infectado", seria Antivírus).

2. Natureza do Firewall

A banca pode perguntar se o Firewall pode ser um equipamento físico.

●     Resposta: SIM. Existe o Firewall de Software (como o do Windows) e o Firewall de Hardware (equipamentos dedicados em grandes redes como as da SEDUC).

3. Falsa Sensação de Segurança

A IBADE pode sugerir que "ter um antivírus instalado dispensa a necessidade de um Firewall".

●     Resposta: Falso. Eles são complementares. O Firewall impede a invasão; o Antivírus combate o invasor caso ele consiga entrar por outros meios (como um Pen Drive).

Grave a analogia da casa:

●     O Firewall é o muro e o portão da casa (controla quem entra e sai pela rua/rede).

●     O Antivírus é o segurança interno que circula pelos cômodos conferindo se as pessoas/arquivos que já estão lá dentro são perigosos.

💾 CAPÍTULO 19 – BACKUP E RECUPERAÇÃO DE DADOS

O Backup (Cópia de Segurança) é o único remédio 100% eficaz contra a perda definitiva de dados. Seja por falha física do computador, exclusão acidental ou ataque de Ransomware (sequestro de dados), ter um backup atualizado é o que separa o caos da continuidade do trabalho.

19.1 Conceito e Importância do Backup

Fazer um backup consiste em realizar a cópia de dados de um local para outro, de modo que possam ser restaurados em caso de perda do original.

Por que o backup é vital para o servidor público?

●     Falha de Hardware: HDs e SSDs têm vida útil limitada e podem quebrar sem aviso.

●     Erro Humano: Excluir acidentalmente uma pasta de notas ou formatar o dispositivo errado.

●     Ameaças Digitais: Vírus que apagam arquivos ou Ransomwares que os bloqueiam.

●     Desastres Físicos: Roubo de equipamentos da escola, incêndios ou picos de energia que queimam componentes.

19.2 Tipos de Backup (O que a IBADE ama cobrar)

Existem diferentes formas de realizar cópias, e a banca adora cobrar a diferença técnica entre elas, focando em tempo de execução e espaço ocupado.

Tipo de Backup

Descrição

Vantagem

Desvantagem

Completo (Full)

Copia todos os arquivos selecionados, independentemente de terem mudado ou não.

Restauração mais rápida e simples (está tudo em um só lugar).

Mais demorado para fazer e ocupa muito espaço.

Incremental

Copia apenas os arquivos que foram criados ou alterados desde o último backup (seja ele completo ou incremental).

É o mais rápido para realizar e ocupa pouco espaço.

A restauração é a mais lenta, pois exige o último backup completo + todos os incrementos.

Diferencial

Copia apenas os arquivos que foram criados ou alterados desde o último backup COMPLETO.

Restauração mais rápida que o incremental (exige apenas o completo + o último diferencial).

Com o tempo, o arquivo de backup vai ficando maior que o incremental.

19.3 Estratégias e Periodicidade

Não adianta fazer um backup hoje e o próximo daqui a um ano. A periodicidade depende da frequência com que os dados mudam. Para a SEDUC, recomenda-se backups diários ou semanais.

A Regra 3-2-1 (Padrão Ouro em Concursos)

Esta é a estratégia de segurança recomendada mundialmente e que cai com frequência em provas:

●     3 Cópias: Tenha pelo menos três cópias dos seus dados (o original + duas cópias).

●     2 Mídias Diferentes: Armazene as cópias em tipos de mídia diferentes (Ex: um HD Externo e um Pen Drive, ou SSD e Nuvem).

●     1 Cópia Off-site: Tenha pelo menos uma das cópias fora do local físico onde o computador original está (Ex: Na nuvem ou em outra escola).

19.4 Recuperação de Dados (Restore)

A recuperação (ou Restore) é o processo inverso do backup: é trazer os dados da cópia de segurança de volta para o computador original ou para um novo dispositivo.

Pontos importantes para a prova:

  1. Integridade do Backup: Um backup só é útil se for testado. A banca pode perguntar sobre a importância de validar os backups.
  2. Mídias de Armazenamento:

○     HD Externo / SSD: Rápido, mas sujeito a falhas físicas.

○     Nuvem (Cloud): Altamente seguro e acessível, mas depende de internet.

○     LTO (Fitas): Usadas em grandes servidores da SEDUC para armazenamento de longa duração.

📌 Como a IBADE cobra este capítulo?

1. Diferenciação de Tipos

"O professor deseja um backup que seja o mais rápido possível de ser executado diariamente, ocupando o mínimo de espaço."

●     Resposta: Backup Incremental.

2. Local de Armazenamento

A banca pode sugerir que "manter o backup em uma partição diferente do mesmo HD é seguro".

●     Resposta: Errado. Se o HD quebrar fisicamente, as duas partições serão perdidas. O backup deve estar em um dispositivo físico diferente.

3. A Regra 3-2-1

Questões que perguntam quantos locais e quantas mídias são necessários para uma estratégia robusta.
